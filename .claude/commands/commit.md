스테이징된 변경사항을 분석하고 한국어 커밋 메시지를 생성해주세요.

1. `git diff --cached` 명령어로 스테이징된 변경사항을 확인하세요
2. 변경사항이 없으면 "스테이징된 변경사항이 없습니다"라고 알려주세요
3. 변경사항이 있으면 아래 규칙에 따라 커밋 메시지를 생성하세요

## 커밋 메시지 규칙

### 형식

```
type(scope): 제목 (50자 이내)

- 변경사항 1
- 변경사항 2
- 변경사항 3
```

### Type 종류

- feat: 새 기능 추가
- fix: 버그 수정
- refactor: 코드 리팩토링
- chore: 빌드, 설정 등 기타 작업
- docs: 문서 변경
- test: 테스트 추가/수정
- style: 코드 포맷팅

### Scope 예시

- api: 백엔드 API 관련
- front: 프론트엔드 관련
- ui: UI 컴포넌트 관련
- auth: 인증 관련
- db: 데이터베이스 관련
- core: 핵심 로직 관련
- config: 설정 관련

### 작성 원칙

- 제목은 50자 이내, 명령형 어조 (~추가, ~수정, ~제거)
- 서술형(~했습니다, ~함) 표현 사용 금지
- 한글 메시지, 영어 키워드(type, scope) 조합
- 변경 목적만 간결하게
- 본문에는 주요 변경사항을 bullet point로 나열

### 예시

```
feat(api): JWT 토큰 검증 추가

- 토큰 만료 시간 검증 로직 추가
- 리프레시 토큰 발급 엔드포인트 구현
- 인증 미들웨어에 토큰 검증 적용
```

```
fix(ui): 다크모드 전환 시 색상 깨짐 수정

- 테마 변경 시 CSS 변수 즉시 적용되도록 수정
- 버튼 hover 상태 색상 누락 보완
```

```
refactor(core): 데이터 캐싱 구조 개선

- Redis 캐시 레이어 분리
- 캐시 무효화 전략 변경 (TTL → 이벤트 기반)
```

## 출력 형식

커밋 메시지를 마크다운 코드 블록으로 출력해주세요:

```
type(scope): 제목

- 세부 변경사항 1
- 세부 변경사항 2
```

git commit 명령어는 실행하지 마세요.
